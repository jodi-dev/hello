Hello World

Posted on 2021-03-03 Edited on 2021-03-06

As per tradition, this will be a tutorial on how to get set up with a Hexo blog, deploy to GitHub pages, install the NexT theme, further customizations and blog writing bread-and-butter workflow.

Hexo Installations

Install Hexo package and initiate default project folder. Prerequisites may include Node.js, npm.

Add Existing Project to GitHub

Prior to pushing project to GitHub, ensure an account has been set up.

Create a public repository but do not initiate with files or templates.

This pushes the default project folder to your GitHub repository.

Enable GitHub Pages

Enable GitHub Pages in the settings tab of your repository. See detailed guide here.

GitHub Deployment Configurations

Modify _config.yml in your project’s root repository to allow deployment to GitHub.

Change the site URL. My blog repo is a sub-directory located inside my GitHub organization. I did not want to use jodi-dev.github.io for this blog, so I set the root as ‘/hello/‘; else, remove the root key or set root as /

Also set deployment type and link to the repository.

Deployment by Hexo command will push to the repository on your behalf. I’m still working on understanding the nuances of using Git tracking and letting Hexo deployment commit automatically - there may likely be merge conflicts if not managed carefully. For now, do not git push to your repository but use the command hexo deploy only.
